## Model Artifacts

All artifacts are generated by running `ML_Working.ipynb` top-to-bottom. The final cell
saves them to `models/`.

| File | Size | Purpose |
|------|------|---------|
| `feature_order.json` | ~1 KB | Column ordering (58 CIC-IDS2017 features) |
| `scaler.joblib` | ~3 KB | StandardScaler fitted on training data |
| `pca.joblib` | ~25 KB | PCA transform (dimensionality reduction) |
| `kmeans.joblib` | ~81 KB | KMeans clustering (assigns flows to clusters) |
| `iso_models.joblib` | ~13 MB | Per-cluster Isolation Forest models |
| `lstm_autoencoder.pt` | ~240 KB | LSTM Autoencoder weights (PyTorch) |
| `xgb_classifier.joblib` | ~444 KB | XGBoost classifier (reserved for future use) |
| `platt_calibrator.joblib` | ~1 KB | Platt scaling calibrator (reserved) |

## Scoring Pipeline

```
Input flow features (dict)
    │
    ▼
┌──────────────────────────────┐
│  1. Feature Alignment        │  Align to feature_order.json (58 cols)
│  2. StandardScaler           │  Normalize using scaler.joblib
│  3. PCA Transform            │  Reduce dimensions via pca.joblib
└──────────┬───────────────────┘
           │
     ┌─────┴─────┐
     ▼           ▼
┌─────────┐ ┌──────────┐
│  IF     │ │  LSTM    │
│ Score   │ │  Score   │
│ (0.4w)  │ │  (0.6w)  │
└────┬────┘ └────┬─────┘
     │           │
     ▼           ▼
┌──────────────────────┐
│  Weighted Ensemble   │  hybrid = 0.4*IF + 0.6*LSTM
│  Composite Confidence│  conf = 0.4*IF_conf + 0.6*LSTM_conf
└──────────────────────┘
           │
           ▼
     { score, confidence, cluster_id, suppressed }
```

### Isolation Forest (IF)

- Uses per-cluster models from `iso_models.joblib` (KMeans assigns cluster first)
- `decision_function()` returns raw anomaly score; sigmoid maps to 0-1
- Confidence: `|raw| / (|raw| + 1)` — farther from boundary = more confident

### LSTM Autoencoder

- Trained on 20-timestep sequences but deployed for single-event scoring
- Reconstruction MSE is compared against `RECONSTRUCTION_THRESHOLD` (default: 140.0)
- Higher reconstruction error = more anomalous
- Confidence: distance from midpoint threshold, normalized to 0-1

### Suppression Gate

- `confidence < CONFIDENCE_THRESHOLD (0.15)` → result is `suppressed: true`
- Low-confidence detections are flagged but not actionable
- Prevents false positives from ambiguous inputs

## Verified Test Results

| Input Type | Score | Confidence | Suppressed | Cluster |
|-----------|-------|------------|------------|---------|
| Benign HTTP | 0.5254 | 0.081 | true | 0 |
| DDoS Attack | 0.8308 | 0.695 | **false** | 0 |
| Zero/Empty | 0.2109 | 0.579 | false | 3 |

Key observations:
- Anomalous traffic scores **58% higher** than benign (0.83 vs 0.53)
- Anomalous traffic has **8.5x higher confidence** (0.695 vs 0.081)
- KMeans correctly assigns different inputs to different clusters (0 vs 3)
- Benign traffic is correctly suppressed (low confidence)

## Backend Integration

### Configuration

Set `ML_SERVICE_URL` in `backend/.env`:
```
ML_SERVICE_URL=http://localhost:8001
```

In Docker Compose, this is set automatically:
```yaml
ML_SERVICE_URL: http://ml-service:8001
```

### Endpoints

| Endpoint | Backend Route | ML Service Route | Description |
|----------|-------------|------------------|-------------|
| ML Health | `GET /api/ml/health` | `GET /health` | Backend proxies to ML service |
| Score Flow | Internal only | `POST /score` | Called by `backend/app/services/ml_service.py` |

### Fallback Behavior

When `ML_SERVICE_URL` is unset or the ML service is unreachable:
- `get_ml_threat_score()` returns `{score: 0.0, confidence: 0.0, suppressed: true}`
- `/api/ml/health` returns a stub response with `model_version: "1.0.0-stub"`
- BWVS calculation proceeds without ML component (ml_threat = 0.0)

## Environment Variables

### ML Service (`ml_service/.env.example`)

| Variable | Default | Description |
|----------|---------|-------------|
| `MODEL_PATH` | `./models` | Path to model artifacts directory |
| `IF_WEIGHT` | `0.4` | Isolation Forest weight in ensemble |
| `LSTM_WEIGHT` | `0.6` | LSTM weight in ensemble |
| `CONFIDENCE_THRESHOLD` | `0.15` | Below this → suppressed |
| `RECONSTRUCTION_THRESHOLD` | `140.0` | LSTM MSE normalization denominator |

### Backend

| Variable | Default | Description |
|----------|---------|-------------|
| `ML_SERVICE_URL` | `""` (disabled) | URL of ML microservice |

## Calibration Notes

The `RECONSTRUCTION_THRESHOLD` of 140.0 was determined empirically:

| Input | Raw LSTM MSE | Normalized Score (MSE/140) |
|-------|-------------|---------------------------|
| Zero vector | 2.64 | 0.019 |
| Benign HTTP | 70.19 | 0.501 |
| DDoS attack | 1.59 × 10¹² | 1.000 (capped) |

The LSTM was trained on 20-timestep sliding windows but serves single events
(seq_len=1) at inference time. This causes inherently higher reconstruction error
for all inputs. The threshold of 140.0 places benign traffic at ~0.5 (midpoint),
ensuring anomalous traffic still saturates at 1.0 while benign stays moderate.

## Docker Deployment

```bash
# Build and run all services
docker compose up -d

# Or build ML service alone
docker compose build ml-service

# Verify ML service health (from backend container)
docker compose exec backend curl -s http://ml-service:8001/health
```

The ML service is internal-only (no exposed ports). The backend communicates with it
via Docker's internal network using service name `ml-service`.

## Graceful Degradation

The ML service loads models at startup and degrades gracefully:

| Models Available | Behavior |
|-----------------|----------|
| IF + LSTM | Full hybrid scoring (default) |
| IF only | Score = IF score, confidence = IF confidence |
| LSTM only | Score = LSTM score, confidence = LSTM confidence |
| None | Service returns error; backend falls back to zero score |

## Known Limitations

1. **scikit-learn version mismatch**: Models trained with sklearn 1.6.1 but loaded
   with 1.8.0. Produces `InconsistentVersionWarning` but works correctly.

2. **Single-event LSTM inference**: The LSTM was trained on 20-event sequences.
   Single-event scoring has inherently higher reconstruction error, mitigated by
   the calibrated `RECONSTRUCTION_THRESHOLD`.

3. **No XGBoost integration yet**: `xgb_classifier.joblib` and `platt_calibrator.joblib`
   are saved but not used in the current scoring pipeline. They can be added as a
   third ensemble component in the future.
